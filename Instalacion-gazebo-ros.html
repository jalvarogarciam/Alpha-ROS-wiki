Instalación de Gazebo y uso del mismo

Guía paso a paso para instalar y utilizar Gazebo con ROS

¿Qué es Gazebo?

Gazebo es un entorno simulado que permite el uso de gemelos digitales de distintos robots en una serie de mundos a tu elección, con la posibilidad de crear también tu propio mundo simulado.

Es ideal para la prueba de nuevas funciones para los robots que tengan su gemelo digital disponible, ya que esto nos permitirá trabajar si no tenemos acceso al robot físico y, por tanto, un avance de forma remota.

Es la mejor opción para introducirse en el mundo de la róbotica, ofreciendo muchas posibilidades de manera totalmente gratuita.

Instalación de Gazebo

Una vez ya tengamos instalado ROS Noetic, el siguiente paso será instalar Gazebo y sus paquetes de simulación. Para ello necesitaremos ejecutar unos comandos sencillos desde la terminal.

1. Instalación de Gazebo

$ sudo apt install ros-noetic-gazebo-ros-pkgs ros-noetic-gazebo-ros-control -y

Explicación: Instala los paquetes esenciales de Gazebo y los controladores necesarios para integrarlo correctamente con ROS Noetic.

Este comando permitirá instalar Gazebo en caso de que no se haya instalado junto a ROS Noetic. Si ya estuviese instalado, únicamente añadirá los plugins necesarios.

Una vez instalado Gazebo, debemos instalar un robot de pruebas para este entorno. En este caso utilizaremos un Turtlebot3 modelo Burger, ya que dispone de sensores de escaneo y es el modelo más utilizado.

2. Instalación de Turtlebot3

$ sudo apt install ros-noetic-turtlebot3 -y

Explicación: Descarga e instala los paquetes principales del robot Turtlebot3 para poder utilizarlo dentro del sistema.

Con este comando se instalarán los archivos básicos de Turtlebot3, aunque todavía no se ha definido el modelo concreto que se va a utilizar. Antes de ello, es necesario instalar el paquete completo de simulaciones.

3. Instalación del paquete de simulación de Turtlebot3

$ sudo apt install ros-noetic-turtlebot3-simulations -y

Explicación: Añade los entornos de simulación, mapas y mundos necesarios para probar el Turtlebot3 dentro de Gazebo.

Con esto, ya solo queda especificar el modelo de Turtlebot3 que se va a utilizar.

4. Configuración de Turtlebot3

$ nano ~/.bashrc

Explicación: Abre el editor de texto Nano para modificar el archivo de configuración principal de la terminal.

Accedemos en modo escritura al archivo ~/.bashrc. Nos desplazamos al final del archivo y añadimos la variable de entorno correspondiente al modelo.

5. Modelo Turtlebot3

$ export TURTLEBOT3_MODEL=burger

Explicación: Define una variable de entorno permanente para que ROS sepa que debe utilizar el modelo burger por defecto.

Por último, es necesario recargar el archivo de configuración para aplicar los cambios:

$ source ~/.bashrc

Explicación: Recarga la configuración de la terminal inmediatamente sin necesidad de reiniciar el sistema.

POSIBILIDADES DENTRO DE GAZEBO

En este documento veremos 3 entornos en los que podemos trabajar:

A) ENTORNO VACÍO

Ideal si queremos trastear con el robot, presentarle obstáculos y ver su funcionamiento.

$ roslaunch turtlebot3_gazebo turtlebot3_empty_world.launch

Explicación: Inicia Gazebo con una simulación vacía y carga el robot Turtlebot3 en el centro del mapa.

B) MUNDO ESTÁNDAR CON OBSTÁCULOS

Es el mundo principal de Turtlebot3, incluye paredes, pasillos y objetos sencillos.

$ roslaunch turtlebot3_gazebo turtlebot3_world.launch

Explicación: Carga el escenario oficial de Turtlebot3 que contiene obstáculos geométricos predefinidos.

C) MUNDO TIPO CASA

Simula un entorno doméstico completo. Perfecto para navegación avanzada.

$ roslaunch turtlebot3_gazebo turtlebot3_house.launch

Explicación: Ejecuta una simulación completa de una casa con divisiones y muebles.
