Sistemas de archivos ROS

Navegando y comprendiendo la estructura de archivos en ROS

El sistema de archivos de ROS

El sistema de archivos de ROS puede encontrarse documentado en la wiki oficial de ROS en el siguiente enlace: http://wiki.ros.org/Tutorials/NavigatingTheFilesystem

Para mantener todo organizado, ROS utiliza un sistema de archivos bien definido basado en paquetes. Entender su estructura es fundamental para saber dónde encontrar los archivos y cómo organizar tu propio código.

  • Paquetes (Packages): Son la principal unidad de organización del software en ROS. Un paquete puede contener nodos (ejecutables), librerías, archivos de configuración, mensajes, servicios y todo lo necesario para que una pieza de software funcione.
  • Package Manifest (package.xml): Cada paquete debe incluir este fichero. Contiene los metadatos del paquete: su nombre, versión, autor, licencia y, lo más importante, sus dependencias con otros paquetes.

Comandos para la Navegación

ROS proporciona una serie de comandos de terminal para moverte por el sistema de archivos de forma eficiente sin necesidad de conocer las rutas exactas.

  • rospack find [nombre_del_paquete]: Devuelve la ruta completa al directorio de un paquete. Es muy útil para saber dónde está instalado algo. 

    
rospack find [nombre_del_paquete]
  • roscd [nombre_del_paquete]:: Cambia el directorio actual de la terminal directamente a la carpeta del paquete especificado. Es el comando que más usarás para navegar a tu código. 

    
roscd [nombre_del_paquete]
  • rosls [nombre_del_paquete]: Lista los ficheros que hay dentro del directorio de un paquete sin necesidad de navegar hasta él. 

    
rosls [nombre_del_paquete]
